Homework 2: Build a “Claw Agents Playground” + Deploy It

The interesting part of agent systems isn’t the chatbot — it’s multiple agents coordinating through shared infrastructure (APIs, protocols, social spaces, games). Platforms like MoltBook made that idea legible: lots of independent agents posting/interacting in a shared environment that looks like a social network.

This homework builds the core muscle of agent infrastructure: designing a small backend and teaching agents how to use it through a SKILL.md so they can actually act together. In lecture, we talked about how this coordination happens in practice mainly through SKILL.md (how an agent learns to use tools and endpoints) and HEARTBEAT.md (how an agent becomes proactive and checks or acts on a schedule).

What you build

A small deployed web app that supports at least 2 “claws” (agents) interacting in a shared space.

Your app must include:

  1. A backend API 

  2. A deployed frontend or simple web page that lets humans watch what’s happening (can be minimal)

  3. A SKILL.md that teaches agents how to use your API endpoints (format should follow the skills pattern OpenClaw uses)

You can build using tools like Cursor / Claude Code, and deploy on platforms like Railway, Google Cloud, Render, etc.

Core requirement: Agents must do something together

Your deployed site must demonstrate:

  • Two agents (ideally two classmates’ agents) successfully using your API to do a shared activity.

  • The activity must be visible in the UI (feed/log/scoreboard/chat history).

Examples of acceptable “together” behaviors:

  • A social feed where agents post updates and reply to each other (lightweight MoltBook vibe).

  • A mini PR/marketing game where agents compete to write the best post and users upvote; the backend tracks rounds/scores.

  • A game loop (e.g., guessing game, trivia duel, dungeon room co-op) where agents take turns via API calls.

  • A “Meet your classmates” wall where each agent posts an intro + tags skills/interests; agents can “ping” each other and form teams.

Be creative, but keep the surface area small.

Deliverables

  1. A link to the deployed website

  2. A 30 to 60 second screen-recorded demo showing agents interacting

Please put everything in a single document.

References and notes

Example GitHub repository:
https://github.com/mariagorskikh/homework2_example.git

Assignment instructions PDF:
https://github.com/mariagorskikh/homework2_example/blob/main/assignment.pdf

Example demo: https://clawmatch.up.railway.app/ 

You may work alone or in teams of two so you can test agent interaction. If you work in a team, please both submit your assignment .

You may use any framework or tooling you like. The important constraint is this: you are not spinning up agents for people. Other students’ OpenClaw agents should be able to join your app. Your system should be genuinely agentic, a shared space where agents communicate, work, or play together, not a scripted demo.

This matters because Homework 1 teaches what an agent is. Homework 2 teaches agent infrastructure. And in Homework 3, you will grow this system and try to attract as many agents as possible by advertising it in this class discussion board (or beyond!)

Have fun with it!
